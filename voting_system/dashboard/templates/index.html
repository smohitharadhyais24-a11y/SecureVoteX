IoT Secure Smart Voting System

Evaluating the cryptographic security and visual analytics layers of SecureVoteX™.

Problem Statement

Traditional smart voting dashboards and voting booths lack verifiable end-to-end audit security, leaving communication channels vulnerable to packet tampering, duplicate voting, and malicious replay attacks.

Project Objectives
  • Provide hardware-enforced multi-factor authentication (RFID + fingerprint matching).
  • Implement encrypted MQTT over TLS telemetry transport pipelines.
  • Enforce message authenticity and non-repudiation using HMAC-SHA256 signatures.
  • Mitigate session-hijacking and replay exploits with chronological sequence numbers.
  • Establish offline ballot buffering with automatic network recovery syncs.
Core Cybersecurity Innovations
HMAC Verification

Payloads are verified using a timing-safe HMAC-SHA256 key, ensuring vote tamper resistance.

Replay Protection

Sequence numbers track the index count of submissions, blocking duplication attempts.
